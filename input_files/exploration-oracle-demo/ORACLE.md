# Oracle 说明（仅评测用）

本题目用于**检测 oracle**：验证求解器是否通过高探索过程找到了唯一认可的正确答案。

## Oracle 定义

- **正确输出**：`/app/result.txt` 内容为单行 `OK:<checksum>`（含换行亦可，strip 后比较）。`<checksum>` 由程序在正确配置下根据配置内容计算，为 8 位小写十六进制，**不在此处写死具体值**（防答案泄露）。
- **正确配置**（供参考，评测不强制提交 config）：  
  `algorithm="bisect"`, `tolerance` 为 (0,1] 内浮点数, `max_iter` 为 ≥10 的整数, `output_mode="checksum"`。

## 探索路径（参考）

1. 运行 `/app` 下程序（如 `python3 /app/runner.pyc`）→ 得到非零退出码或错误日志。
2. 假设某配置项错误（如 `algorithm`）→ 修改 config → 再运行 → 观察。
3. 重复假设-验证直至四个关键项均正确，程序写入 `OK:<checksum>`。
4. 评测时仅检查 `result.txt` 是否与 verifier 根据正确配置计算的期望一致，从而**检测是否找到 oracle**。

## 高探索空间对应关系

| 维度           | 本题体现                                       |
|----------------|------------------------------------------------|
| 假设-验证循环  | 多次改 config → 运行 → 看 stderr/退出码/result |
| 决策点数量     | 4 个关键配置项，顺序与组合均可尝试             |
| 可验证中间结果| 每次运行都有退出码与 stderr                    |
| 跨领域         | JSON 配置 + 程序语义 + 日志分析                |
| 错误定位       | 日志不指名具体 key，需推断                     |
| 解的质量       | 仅“与 oracle 一致”为通过                      |

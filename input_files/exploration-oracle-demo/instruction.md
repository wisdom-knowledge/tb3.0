# 配置诊断与 Oracle 验证（高探索空间）

## 背景

`/app` 下有一个依赖配置文件的数值求解器（通过运行该程序执行）。当前环境下程序会**失败或输出错误结果**，且错误信息不会直接指出是哪一个配置项有问题。

你的目标：通过**观察日志、提出假设、验证、迭代**，找到使程序正确运行并产生标准结果的配置，并将**正确结果**写入 `/app/result.txt`。

## 要求

1. **不要**假设只有一个配置项错误，可能需要同时修正多个项。
2. 程序从 `/app/config.json` 读取配置，运行后应把**唯一认可的正确答案**写入 `/app/result.txt`。
3. 正确结果格式为**一行**：`OK:<checksum>`（末尾可有换行）。其中 `<checksum>` 为** 8 位小写十六进制字符串**，由程序在正确配置下根据配置内容计算得出（例如形如 `a7f3c2e9`）。判题时以 strip 后与期望一致为准。
4. 你可以多次修改 `/app/config.json` 并重新运行该程序，通过退出码和 stderr 判断当前假设是否成立。

## 约束

- 仅允许修改 `/app/config.json` 或由程序生成的结果文件；**不要修改 /app 下的程序或字节码**。
- 最终判定以 `/app/result.txt` 的内容是否与 oracle 期望一致为准。

## 探索空间说明（供评估参考）

本题目具有高探索空间特征：

- **假设-验证循环**：需要多次“改配置 → 运行 → 看日志/结果”的循环。
- **决策点**：多个配置项（算法、容差、迭代上界、输出模式等）均可能影响行为。
- **可观察中间结果**：每次运行都有退出码、stderr 及可能的 result.txt 内容。
- **跨知识点**：需结合配置文件格式、程序语义与日志分析。
- **错误定位**：需根据日志/状态推断错误配置，而非依赖单一明确报错信息。
